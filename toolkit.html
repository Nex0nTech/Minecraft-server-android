NexonTech Android Toolkit – Menu Structure Explained

The NexonTech Android Toolkit is a Windows batch utility designed to simplify ADB and Fastboot operations. Instead of typing commands manually, the user interacts with a clean, color‑coded menu. Below is the exact structure of the menu as it appears inside the toolkit, followed by a detailed explanation of what each option does.

What Each Menu Option Does

1) Install SDK Platform Tools

Downloads and installs the official Android Platform Tools into the user’s Downloads folder. This enables all ADB and Fastboot operations. The toolkit automatically enters the platform-tools directory after installation.

2) Check ADB connection

Verifies whether the connected device is detected by ADB. Useful for confirming USB debugging and driver functionality.

3) Reboot to bootloader

Reboots the device into fastboot/bootloader mode. Required before flashing partitions or performing low‑level operations.

4) Reboot to recovery

Reboots the device into recovery mode (stock or custom).

5) Install APK from PC

Installs an APK stored on the user’s computer directly onto the device. The toolkit asks for the file path and handles the installation.

6) Uninstall package (debloat)

Removes a package (app) from the device for the current user. Useful for debloating system apps, but removing the wrong package may break functionality.

7) List installed packages

Displays all installed package names on the device. Often used before debloating to identify the correct package name.

8) Pull file from device

Copies a file from the device to the PC. The toolkit asks for the source path on the device and the destination path on the computer.

9) Push file to device

Copies a file from the PC to the device. The user specifies both the source and destination paths.

10) USB debugging instructions

Shows step‑by‑step instructions on how to enable USB debugging on Android.

11) Check Fastboot connection

Verifies whether the device is detected in fastboot mode. Essential before flashing a stock ROM.

12) Flash boot.img

Flashes the boot partition using a boot image provided by the user. Critical operation: using the wrong file may cause bootloops.

13) Flash recovery.img

Flashes the recovery partition. Can install stock recovery or a custom one. Incorrect images may brick the device.

14) Extract boot.img from payload.bin

Extracts images (such as boot.img) from a payload.bin file found in many modern ROMs. Useful for patching or flashing.

15) Show device info

Displays detailed device information such as build properties, hardware info, and system configuration.

16) Clear ADB keys

Deletes ADB authorization keys on the PC and restarts the ADB server. Forces the device to request authorization again.

17) Restart ADB server

Restarts the ADB server without deleting keys. Useful when ADB stops detecting the device.

18) Exit

Closes the toolkit and returns to the Windows desktop.
